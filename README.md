# Software Engineering - Github Collaboration Practice

This repository is used for practicing code collaboration using GitHub.

## Practice Goals

Students are capable of:
1. Creating a branch corresponding feature task.
2. Committing and pushing to GitHub.
3. Creating a Pull Request.
4. Reviewing codes to other team's PR.
5. Merging after getting approval.

## Case Study

Simple To-Do List app for managing software engineering project tasks.

## Features

- Show overall tasks
- Add new tasks
- Change task status
- Delete tasks
- Search tasks by assignee

## Practice Workflow

1. Clone repository.
2. Create a new branch.
3. Working on features corresponding to each task.
4. Commit changes.
5. Push to GitHub.
6. Make a pull request.
7. Review others' pull request.
8. Fixing code if there are any comments.
9. Repeat from step 7 and 8 until the pull request is approved.
10. Once approved, sync the changes with the main branch.
